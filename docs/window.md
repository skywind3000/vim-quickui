# QuickUI Window Module Reference

> **File**: `autoload/quickui/window.vim`
> **Dependencies**: `quickui#core`, `quickui#utils`
> **Requires**: Vim 8.2+ (popup window) / Neovim 0.4+ (floating window)

## Overview

`quickui#window` is QuickUI's low-level window abstraction layer, wrapping Vim's `popup_*` API and Neovim's `nvim_open_win` floating window API into a unified object-oriented interface. Upper-layer components (such as the `input.vim` input box) use this module to create and manage floating windows without worrying about platform differences.

### Design Role

- **Not** a user-facing component, but rather QuickUI's internal infrastructure layer
- Provides unified window lifecycle management: create, show/hide, move, resize, close
- Provides buffer content management and syntax highlighting helpers
- Automatically handles border emulation on Neovim (early Neovim versions lack built-in border support)

## Architecture

### Vim vs Neovim Difference Abstraction

| Capability | Vim Implementation | Neovim Implementation |
|---|---|---|
| Window creation | `popup_create()` | `nvim_open_win()` |
| Show/hide | `popup_show()` / `popup_hide()` | Close and recreate window |
| Window move | `popup_move()` | `nvim_win_set_config()` |
| Resize | `popup_move()` with min/max width/height | `nvim_win_set_config()` |
| Border | `popup_setoptions()` built-in border support | Separate background window to emulate border |
| Close callback | `popup_setoptions()` callback | None (manual management) |
| Close button | `popup_setoptions()` close='button' | Draw close button character in border buffer |
| Command execution | `quickui#core#win_execute()` | `quickui#core#win_execute()` |

### Neovim Border Emulation

Early Neovim does not support native floating window borders, so window.vim uses a "dual window" approach:

1. **Foreground window** (`self.winid`): Displays actual content, size is `w x h`
2. **Border window** (`self.info.border_winid`): Displays border characters as a background window, size is `tw x th` (including border and padding), z-index is 1 less than the foreground window

Border content is generated by `quickui#utils#make_border()` as text lines, written to a separate buffer.

## Class Definition: `s:window`

### Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `w` | Number | 1 | Content area width (characters) |
| `h` | Number | 1 | Content area height (lines) |
| `x` | Number | 1 | Window top-left column coordinate (0-based, relative to editor) |
| `y` | Number | 1 | Window top-left row coordinate (0-based, relative to editor) |
| `z` | Number | 40 | Priority/z-index (actual z-index is `z+1`, border is `z`) |
| `winid` | Number | -1 | Current window ID (-1 means not opened or hidden [Neovim]) |
| `dirty` | Number | 0 | Whether buffer content needs updating |
| `text` | List | [] | Current buffer text line list |
| `bid` | Number | -1 | Content buffer ID (allocated via `quickui#core#buffer_alloc()`) |
| `hide` | Number | 0 | Whether in hidden state (0=visible, 1=hidden) |
| `mode` | Number | 0 | Lifecycle state (0=not created/closed, 1=created) |
| `opts` | Dict | {} | Deep copy of creation options |
| `info` | Dict | {} | Internal environment data computed during initialization |
| `quit` | Number | 0 | Whether closed by close button (Vim only, set via callback) |

### `info` Dictionary Internal Fields

| Field | Description |
|-------|-------------|
| `tw` | Total width (including padding + border) |
| `th` | Total height (including padding + border) |
| `has_padding` | Whether padding is present |
| `has_border` | Whether border is present |
| `cmd` | List of initialization commands to execute after window creation |
| `pending_cmd` | Commands cached while window is not yet shown |
| `border_winid` | Neovim border window ID (-1 means none) |
| `border_bid` | Neovim border buffer ID (-1 means none) |
| `nvim_opts` | Neovim foreground window `nvim_open_win()` options |
| `border_opts` | Neovim border window `nvim_open_win()` options |
| `border_init` | Neovim border window initialization commands |
| `sim_border` | Whether Neovim uses simulated border |
| `off_x` / `off_y` | Neovim foreground window offset from logical position (x,y) (caused by border+padding) |
| `syntax_cmd` | Syntax highlighting commands collected during `syntax_begin/end` |
| `syntax_mod` | Syntax highlighting mode flag |

## Constructor

### `quickui#window#new()`

Creates and returns a new window object instance (a deep copy of `s:window`).

```vim
let win = quickui#window#new()
```

The returned object is not yet associated with any window — you must call `win.open()` to create an actual popup/floating window.

## Method Reference

### Lifecycle Methods

#### `win.open(textlist, opts)`

Creates and displays a window. If the window is already open, it first calls `close()` to close the old window.

**Parameters**:

- `textlist`: String or List. Initial window content. Strings are split by `\n` into a line list.
- `opts`: Dict. Creation options — see [Options Reference](#options-reference) below.

**Flow**:
1. Calls `close()` to ensure the old window is closed
2. Calls `__prepare_opts()` to parse options, allocate buffer, calculate dimensions
3. Calls `__vim_create()` or `__nvim_create()` depending on platform
4. If `opts.center` is true, auto-centers the window
5. If `opts.hide` is 0 (default), immediately shows the window

#### `win.close()`

Closes the window and releases all resources.

**Behavior**:
- Vim: Calls `popup_close()`
- Neovim: Calls `nvim_win_close()` for both content and border windows
- Releases content buffer and border buffer (`quickui#core#buffer_free()`)
- Resets `winid=-1`, `bid=-1`, `mode=0`, `hide=0`

#### `win.show(show)`

Controls window visibility.

**Parameter**:
- `show`: Number. `1` to show, `0` to hide.

**Platform differences**:
- Vim: Calls `popup_show()` / `popup_hide()` (window ID remains unchanged)
- Neovim: Hiding closes the window (`nvim_win_close`), showing recreates it (`nvim_open_win`), so **on Neovim `winid` changes after hide/show**

### Position and Size Methods

#### `win.move(x, y)`

Moves the window to the specified position, automatically clamping to screen boundaries.

**Parameters**:
- `x`: Column coordinate (0-based)
- `y`: Row coordinate (0-based)

**Boundary clamping logic**:
- If `x + tw > &columns`, then `x = &columns - tw`
- If `y + th > &lines`, then `y = &lines - th`
- x and y are clamped to no less than 0

Note: `tw`/`th` is the total size including border and padding.

#### `win.center(...)`

Centers the window on screen.

**Parameter**:
- `style` (optional, default 0):
  - `0`: Top-biased centering, uses 38% of screen height as vertical center (more visually natural)
  - `1`: Standard centering, uses the center of 80% of the screen area

#### `win.resize(w, h)`

Resizes the window content area.

**Parameters**:
- `w`: New content width
- `h`: New content height

**Behavior**:
- Updates `self.w`, `self.h` and `info.tw`, `info.th`
- Vim: Uses `popup_move()` to set min/max width/height
- Neovim: Uses `nvim_win_set_config()`, also rebuilds border buffer content

### Content Management Methods

#### `win.set_text(textlist)`

Sets all window text content.

**Parameter**:
- `textlist`: String (split by `\n`) or List (used directly).

Immediately calls `update()` to write to the buffer.

#### `win.set_line(index, text [, refresh])`

Sets the text of a specific line.

**Parameters**:
- `index`: Line number (0-based)
- `text`: Line content string
- `refresh` (optional, default 1): Whether to immediately refresh the buffer

If `index` exceeds the current line count, empty lines are automatically padded to reach the required count.

#### `win.get_line(index)`

Gets the text of a specific line. Returns an empty string if out of range.

#### `win.update()`

Writes the `self.text` list to the buffer (`quickui#core#buffer_update()`). Typically called after modifying `self.text`.

### Command Execution Methods

#### `win.execute(cmdlist)`

Executes Ex commands in the window context.

**Parameter**:
- `cmdlist`: String (split by `\n`) or List. Command list to execute.

**Behavior**:
- If the window is open (`winid >= 0`), executes immediately via `quickui#core#win_execute()`
- If the window is not yet shown, commands are appended to `info.pending_cmd` and automatically executed when the window is displayed

This deferred execution mechanism ensures that commands set before window creation are not lost.

### Syntax Highlighting Methods

This group of methods provides a begin/end pattern for batch syntax highlighting, avoiding frequent win_execute calls.

#### `win.syntax_begin([mod])`

Begins a syntax highlighting batch. Initializes the command list and clears existing syntax rules.

**Parameter**:
- `mod` (optional, default 1): Mode flag passed to `quickui#core#high_region()`

#### `win.syntax_region(color, x1, y1, x2, y2)`

Adds a highlight region to the current batch.

**Parameters**:
- `color`: Highlight group name (e.g., `'QuickInput'`, `'QuickCursor'`)
- `x1, y1`: Start position (0-based)
- `x2, y2`: End position (0-based)

Coordinates are automatically incremented by 1 to convert to Vim's 1-based line/column numbers. If y1==y2 and x1>=x2, the call is skipped (empty region).

#### `win.syntax_end()`

Ends the syntax highlighting batch and executes all accumulated highlighting commands.

**Typical usage pattern**:

```vim
call win.syntax_begin()
call win.syntax_region('QuickSel', 0, 2, 10, 2)
call win.syntax_region('QuickKey', 0, 3, 5, 3)
call win.syntax_end()
```

### Mouse Interaction Methods

#### `win.mouse_click()`

Handles a mouse click event, returning the click position relative to the window content area.

**Returns**: Dict `{'x': col, 'y': row}`, coordinates are 0-based. Returns `{'x': -1, 'y': -1}` if the click is not within this window.

Automatically subtracts the border offset when borders are present.

### Other Methods

#### `win.refresh()`

Forces a window display refresh. On Vim, calls `popup_setoptions(winid, {})`, then `redraw`.

## Usage Examples

### Basic Window

```vim
let win = quickui#window#new()
call win.open(['Hello, World!', 'Line 2', 'Line 3'], {
    \ 'w': 30, 'h': 3,
    \ 'x': 10, 'y': 5,
    \ 'border': 1,
    \ 'title': ' Demo ',
    \ 'color': 'QuickBG',
    \ })

" Modify content
call win.set_line(1, 'Updated Line 2')

" Move window
call win.move(20, 10)

" Close
call win.close()
```

### Deferred Command Execution

```vim
let win = quickui#window#new()

" Commands set before window creation are cached
call win.execute('setl cursorline')

" Cached commands are automatically executed when the window is created
call win.open(lines, {'w': 40, 'h': 10, 'hide': 1})

" Show window
call win.show(1)
```

### Syntax Highlighting

```vim
call win.syntax_begin()
call win.syntax_region('QuickSel', 0, 0, 20, 0)   " line 1, columns 0-20
call win.syntax_region('QuickKey', 0, 1, 5, 1)     " line 2, columns 0-5
call win.syntax_end()
```

## Notes

1. **Neovim winid is unstable**: `show(0)` closes the window (winid becomes -1), `show(1)` recreates the window with a new winid. Do not cache winid for long-term use.
2. **Coordinate system**: `x`/`y` use 0-based coordinates, but Vim's popup API uses 1-based coordinates — window.vim handles the +1 conversion internally.
3. **Border dimensions**: `tw`/`th` is the total size including border + padding, `w`/`h` is the pure content area size. `move()`'s boundary clamping is based on `tw`/`th`.
4. **Buffer management**: window uses `quickui#core#buffer_alloc/free` for buffer pool management. Do not manipulate buffers directly, and do not use `bid` after `close()`.
5. **quit property**: Only effective on Vim — set to 1 via the `popup_exit` callback when the user clicks the close button. Neovim has no such mechanism.
6. **mode property**: `0` means the window is not created or has been closed, `1` means created. Most methods return early when `mode==0`.
